# claude-ux.md — 비전문 유저용 인터페이스 제작 가이드

> **언제 읽나**: 사용자가 "유저 친화적으로 개선", "너무 복잡하다", "이해가 안 된다", "전문용어 많다", "직관적이지 않다" 등을 말하면 **이 문서를 먼저 읽고** 그 원칙대로 진행한다. (CLAUDE.md §UX 참조)
>
> **대상 유저**: 마케팅을 운영하지만 통계·데이터 분석에 익숙하지 않은 실무 마케터. 데이터는 준비돼 있으나 통계값·전문용어를 그대로 보면 이해하지 못하고, 전체 분석 과정이 어떻게 흘러가는지도 모른다.
>
> 이 가이드는 5-18(마케팅 반응 분석)의 카니발 진단 UI를 비전문가용으로 재설계하며 확립한 원칙이다. 다른 도구·화면에도 동일하게 적용한다.

---

## 0. 대원칙 — "결론 먼저, 근거는 접어서"

화면을 **2층**으로 나눈다.
- **1층 (메인, 항상 보임)**: 짧은 평어 결론 + 액션. 통계값·전문용어 **없음**.
- **2층 (토글/아코디언, 기본 접힘)**: 통계 검정·계수·p값·방법론. 원하는 사람만 펼침.

전문가가 원하는 원값은 **버리지 않는다** — 2층에 두거나 흐린 작은 글씨로 강등한다. "숨긴다 = 삭제"가 아니라 "숨긴다 = 위계를 낮춘다".

**네이티브 `title` 속성 단독 사용 금지**: `<span title="...">(마우스오버)</span>`처럼 브라우저 기본 툴팁에만 의존하면, hover 가능하다는 시각적 단서(점선 밑줄·아이콘 등)가 없어 유저가 그 존재 자체를 모른다("전문 수치는 여기 마우스오버"라고 써봤자 발견 안 됨 — 실사용에서 확인된 함정). 원값은 기본적으로 **바로 보이는 흐린 작은 글씨**로 두거나, 정말 hover UI를 쓸 거면 점선 밑줄(`border-bottom:1px dotted`)·`cursor:help`·(i) 아이콘 등 명확한 어포던스를 같이 준다.

## 1. 여정(flow)을 먼저 보여준다

유저는 전체 과정이 어떻게 흘러가는지 모른다. 도구가 여러 단계(탭)면, 각 단계를 **§번호·전문용어가 아니라 "질문"**으로 프레임한다.

- ❌ "① 카니발 진단 — 어느 채널이 오가닉을 잠식하나 / lead-lag" 
- ✅ "① 잠식 진단 — 유료 광고가 공짜 유입을 갉아먹고 있나?"

3툴 예시(5-18):
1. **잠식 진단** — "광고가 공짜 유입을 갉아먹나?"
2. **기여 분해** — "무엇이 성과를 움직였나? 다음 예산 어디에?"
3. **미래 예측** — "이대로 가면, 예산 바꾸면 성과는?"

현재 어느 단계인지 항상 시각적으로(active 하이라이트) 표시.

---

## 2. 그룹핑 > 나열 (칸반 원칙)

여러 항목(채널 등)에 상태가 붙을 때, **각 항목마다 배지를 반복**하면 유저가 하나씩 확인해야 한다. 대신 **상태별 칼럼(칸반)으로 그룹핑**한다 — 칼럼 헤더가 곧 상태.

- ❌ 카드 그리드: 카드마다 [배지] 반복 → "이 마크가 뭐지?"를 계속 확인
- ✅ 칸반: `잠식 의심` / `애매함` / `문제 없음` 3칸, 채널은 칩으로 해당 칸 안에

**버킷 수는 3개 권장** (조치 필요 / 애매 / 괜찮음). 엔진이 5단계·N단계를 내도 마케터용으로는 3개로 접는다. 칩은 이름 + 한 줄 액션만; 상세 숫자는 클릭 시 드릴다운.

색: 위험=red, 애매=neutral/gray, 안전=green (semantic 토큰: `--bg-danger`/`--bg-success`, `BADGE_TONE`).

## 3. 지표·검정은 "질문 + 평어 답"으로

통계 지표 이름을 그대로 쓰지 않는다. **무엇을 묻는지 평어 질문**으로 라벨을 달고, 결과도 평어로.

| ❌ 전문용어 | ✅ 평어 질문 |
|---|---|
| 시간 선행성 (저지출 ≤p25) | 광고를 늘리기 전에 오가닉이 이미 줄고 있었나? |
| 허위상관 (탈추세·차분) | 시즌·추세를 걷어내도 광고 늘 때 오가닉이 주나? |
| 순증분 (net 탄력성) | 광고를 늘리면 전체 성과가 순증가하나? |
| 그랜저 인과 | 광고비가 몇 주 뒤 오가닉을 끌어내리나? |

투표/판정값도 평어로: `FOR/AGAINST/ABSTAIN` → **괜찮음 / 잠식 신호 / 판단 보류**. 원 통계값(slope·p·CI·F)은 카드 하단 흐린 글씨 + `title="통계 원값(전문가용)"`.

전문용어를 꼭 노출해야 하면 **평어 먼저, 괄호로 전문어**(`재현율(Recall)`), 또는 `title` 툴팁.

## 4. 결론과 근거가 **모순돼 보이지 않게**

메인 배지(그룹/헤드라인)와 상세 판정이 **다른 규칙으로 계산되면** 유저는 "문제없다며 왜 의심?"이라고 혼란스러워한다.
- 상세 뷰의 헤드라인은 **메인 그룹핑과 동일한 규칙**으로 계산한다.
- 종합 판정이 개별 지표와 다를 때(예: 같은 주 지표는 OK인데 시차 지표에서 걸림) → **왜 그런지 한 문장으로 평어 설명**:
  > "같은 주 지표(①②③)는 괜찮은데, 몇 주 시차를 두고 광고비가 오가닉을 끌어내리는 신호(④)가 나와서 의심으로 올렸어요."

## 5. 레이아웃 — 균등·정렬

- N개 동급 항목은 `display:grid; grid-template-columns:repeat(N, minmax(0,1fr))`로 **균등 분배**. `flex`는 왼쪽 쏠림("옹기종기")을 만든다 → 지양.
- ①②③는 ①왼쪽 · ②가운데 · ③오른쪽처럼 순서가 위치로 읽히게.
- Card(`border`+`radius:12px`+옅은 shadow)로 논리 단위를 시각적으로 묶는다.
- **범례 많은 차트(회귀 변수 등)**: Chart.js 기본 상단 범례가 캔버스를 짓눌러 레이아웃 붕괴 → `legend.position:"right"`+`maxWidth` + 컨테이너 `minHeight` 명시 + x축 `autoSkip:true, maxTicksLimit` + 툴팁 `toLocaleString()`. `maintainAspectRatio:false, responsive:true`는 기본.
- **아코디언/토글 안의 차트는 접힘 상태에서 폭 0으로 마운트됨**(§7 0px 함정 재발) → `<details onToggle>`에서 열릴 때 `requestAnimationFrame(()=>window.dispatchEvent(new Event("resize")))`로 재측정. 통계 상세를 접어두면 거의 항상 이 문제가 생긴다.

## 6. "더 알고 싶으면" 탈출구

맨 밑에 **전 과정을 상세히 설명하는 문서 다운로드 버튼**을 둔다("이 과정에 대한 자세한 설명이 듣고 싶으신가요? — 상세 문서 받기"). 문서는:
- 평어 먼저(한 줄 요약 → 왜 중요 → 각 신호 평어 설명 → 판정 종합 규칙 → 한계),
- 전문 용어는 각 항목 끝에 `(전문: …)`로 병기,
- **현재 데이터의 판정 요약**을 포함(자기 데이터로 재확인 가능),
- 마크다운(.md) 텍스트 다운로드(`textDownload`).

---

## 7. 정직성 (통계적 겸손) — 절대 훼손 금지

유저 친화적으로 단순화하되 **거짓으로 만들지 않는다**.
- "연관(association)이지 인과(causation) 아님" 고지 유지. 확정은 홀드아웃 실험.
- 데이터 부족·공선이면 억지 판정 대신 **"판단 보류"**로 정직하게.
- 수학 엔진(`*Math.js`)은 절대 건드리지 않는다 — **렌더층만** 재구성(골든 테스트 불변).
